# Public Safety Rules

Even in a career context workspace:

- no employer source code
- no copied internal docs
- no credentials
- no customer-sensitive material
- no confidential diagrams
- no endpoints
- no private implementation details that would turn the repo into an internal system archive

Allowed:

- high-level career context
- user-confirmed metrics
- public links
- generalized descriptions of systems, technologies, ownership, adoption, and impact
- local/internal source references inside `Source Notes` only, when useful for future private maintenance

Final resume artifacts must never contain internal URLs, local source paths, copied internal text, customer data, or secrets.
