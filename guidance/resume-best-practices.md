# Resume Best Practices

Use this guidance when editing the resume.

- Lead with accomplishment bullets, not responsibility-only bullets.
- Prefer action + system + scale + outcome.
- Quantify impact when honest and explain uncertain numbers as estimates.
- Translate internal project names into external descriptions.
- Keep role titles, companies, and dates easy to parse.
- Use keywords naturally, not as stuffing.
- Keep one page by default.
- Favor concrete production signals: shipped, adopted, used by teams, reduced time, improved reliability, automated workflow, led design, mentored developers.
- For AI/agentic engineering, emphasize production systems, evaluation, observability, governance, tool use, retrieval quality, human approval flows, and measurable developer or business impact.
